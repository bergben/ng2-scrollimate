Method: percentLeft

This turns blue when 20% or less of the total viewport height is left between the element top and the viewport top.
currentValue: {{scrollimateOptions.percentLeft.currentValue | number}}%

Method: pxLeft

This disappears when 200px or less are left from the top viewport.
currentValue: {{scrollimateOptions.pxLeft.currentValue | number}}px
Please note that for fixed elements it's important to set the option "setAtLastChance: false".

Method: pxTotal

This turns green when a total of 500px of the page has been scrolled.
currentValue: {{scrollimateOptions.pxTotal.currentValue | number}}px

Method: percentTotal

This text becomes bigger and bold when a total of 37.55% of the page has been scrolled.
currentValue: {{scrollimateOptions.percentTotal.currentValue | number}}%

Method: pxElement

This just appeared because the top of the element is 400px or more from the bottom viewport up and has the css class "added-class" added (therefore the border turns red).
currentValue: {{scrollimateOptions.pxElement.currentValue | number}}px

Method: percentElement

This rotates because it has the class "rotating" added once 40% of the element or more are in the viewport from bottom up. Once it reaches 200% of the element it will stop rotating again.
This is an example for multiple states / animations on an elment.

currentValue: {{scrollimateOptions.percentElement.currentValue | number}}%
